# Границы доверия в llm-wiki-agent

Этот документ фиксирует безопасную рамку текущего кэпстоун-проекта. Он не
добавляет новую задачу студенту, а объясняет, какие части проекта уже считаются
доверенными, какие являются входными данными и где агенту пока нельзя менять
состояние.

## Текущий принцип

В базовом пути агент сначала учится безопасно читать:

1. читает небольшой локальный корпус из `vault/raw/`;
2. читает схему в `docs/wiki-schema.md` как правила устройства wiki;
3. компилирует производный wiki-слой в `vault/wiki/`;
4. собирает ограниченный пакет контекста в `artifacts/context/`;
5. пишет ответ и сводку запуска в `vault/outputs/`;
6. оставляет запись запуска, трейсы, контракты инструментов и отчёт проверки состояния в
   `artifacts/`.

Модель может участвовать в подготовке финального ответа, но не получает
самостоятельного права изменять устойчивый слой знаний. Все записи в файловую
систему делает приложение, а не инструмент записи, вызванный моделью.

## Границы слоёв

**`vault/raw/`** — исходный корпус. В текущих майлстоунах это вход только для чтения.
Demo может читать эти файлы, но не должна переписывать их.

**`vault/wiki/`** — скомпилированный wiki-слой. Он создаётся из `vault/raw/` и
пока считается производным артефактом, а не местом для автоматического
обратной записи.

**`docs/wiki-schema.md`** — правила структуры wiki и процедур её поддержки.
В базовом срезе это договорённость только для чтения, которая не меняется во время запуска.

**`vault/outputs/`** — ответы и сводки конкретных запусков. Сюда можно писать
результат запуска, потому что это не меняет базу знаний.

**`artifacts/`** — служебные доказательства запуска: пакет контекста, план,
трейсы, запись запуска, контракты инструментов, отчёт проверки состояния и метаданные от OpenAI
Responses API для live-запусков.
Эти файлы нужны для диагностики, а не для обучения модели на лету.

## Чувствительные Домены

Даже учебный лор Northstar Compute показывает важную production-рамку. В
реальной AI-инфраструктурной компании чувствительными были бы:

1. имена клиентов, условия контрактов и эскалации;
2. планы capacity, GPU-allocation и ограничения поставок;
3. pricing, скидки, маржа и прогнозы выручки;
4. отчёты об инцидентах, reliability-данные и внутренние решения;
5. условия вендоров, roadmap и переговорные позиции;
6. секреты, учётные данные, API-ключи и приватные значения эндпоинтов.

В текущем репозитории эти данные синтетические. Но правило остаётся тем же:
live-путь получает только выбранный пакет контекста, а не весь vault. Если
студент заменяет мок-корпус реальными документами, сначала нужна политика
редактирования чувствительных данных и отдельное согласие на отправку контекста
во внешний API.

## Опциональный live-путь OpenAI

Live-команда отправляет в OpenAI Responses API только выбранный вопрос,
wiki-контекст и исходное основание, которые уже попали в пакет контекста.

Live-путь не даёт модели:

1. доступ к `OPENAI_API_KEY` как к тексту;
2. доступ к shell;
3. доступ к git;
4. инструмент для изменения `vault/raw/`;
5. инструмент для изменения `vault/wiki/`;
6. право выполнять обратную запись без отдельного потока подтверждения.

В текущем основном пути live-режим — это один вызов Responses API для финального
ответа. Здесь нет hosted web search, File Search, computer use, фоновых задач
или постоянного состояния диалога.

Ответ модели сохраняется в `vault/outputs/<run_id>.md`, а технические метаданные —
в `artifacts/responses/<run_id>.json`.

`OPENAI_API_KEY` не должен попадать в трейсы, отчёт проверки состояния, запись запуска или
метаданные ответа. Если в `vault/raw/` лежат приватные или секретные документы,
не запускайте live-режим до отдельной политики редактирования и согласия.

## Будущая обратная запись

Расширение A может добавить контролируемую обратную запись, но только как отдельный
безопасный слой:

1. агент готовит предложение или patch;
2. человек видит предпросмотр;
3. подтверждение сохраняется как часть состояния запуска;
4. запись выполняется приложением;
5. трейс показывает, кто и что разрешил.

До появления этого слоя любые изменения устойчивого wiki-слоя по инициативе модели
считаются вне объёма.

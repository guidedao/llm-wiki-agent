# Контракты инструментов

В этом проекте инструмент — это не просто Python-функция. Инструмент считается
частью агентного runtime, поэтому у него должен быть короткий контракт:

1. **что принимает**;
2. **что возвращает**;
3. **читает или пишет**;
4. **можно ли показывать его модели как инструмент**;
5. **куда он может записать результат**.

Такой контракт нужен не ради формальности. Он помогает не смешивать «модель
выбрала контекст» и «модель получила право менять файлы».

## Текущий слой

Сейчас локальный demo-запуск сохраняет реестр инструментов в
`artifacts/tools/<run_id>.json`.

В нём есть четыре поверхности:

1. **Собрать wiki-слой** — запись приложением в `vault/wiki/`.
2. **Найти wiki-страницы** — поиск только для чтения по скомпилированному wiki-слою.
3. **Прочитать raw-источник** — чтение одного файла из `vault/raw/`.
4. **Записать ответ запуска** — запись приложением в `vault/outputs/`.

Инструменты только для чтения можно позже экспортировать в формат OpenAI
Responses API. Инструменты записи пока не показываются модели как доступные
действия.

В текущем live-пути модель пока не вызывает эти инструменты напрямую. Приложение
само собирает wiki- и raw-контекст, передаёт его в Responses API и сохраняет
ответ. Поэтому `artifacts/tools/<run_id>.json` сейчас работает как проверяемый
контракт поверхности, а не как список активных вызовов инструментов внутри модели.

## Доменные Поверхности На Будущее

Лор Northstar Compute может подсказать будущие инструменты только для чтения, но
они не входят в базовый срез. Возможные расширения:

1. **Профиль кластера** — читать сводку по GPU-кластеру без секретов и
   управляющих действий.
2. **Определение метрики** — объяснять utilization, queue time или маржу по
   зафиксированному источнику.
3. **Профиль клиента** — читать только синтетический или отредактированный клиентский контекст.
4. **Предложение обновления wiki** — готовить предложение без применения patch.
5. **Применение patch после подтверждения** — выполнять контролируемую обратную запись только
   приложением и только после явного подтверждения.

Эти поверхности полезны позже, но базовый срез намеренно остаётся контуром
«сначала читаем»:
студент учится проектировать границы до того, как модель получает активные
действия.

## Почему инструменты записи закрыты

`write_answer_artifact` пишет только ответ конкретного запуска. Это безопаснее,
чем обратная запись в базу знаний, потому что ответ лежит отдельно и не меняет
устойчивый слой знаний.

`compile_wiki_layer` тоже пишет файлы, но делает это как детерминированный этап
приложения: из `vault/raw/` в `vault/wiki/`. Модель не вызывает этот инструмент
самостоятельно.

Будущая контролируемая обратная запись должна быть отдельным слоем: предложение
изменения, предпросмотр, подтверждение, запись приложением и трейс разрешения.

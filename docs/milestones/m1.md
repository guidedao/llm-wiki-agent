# M1

## Цель

В этой вехе wiki-слой становится основной поверхностью поиска, а финальный
ответ можно собрать через OpenAI Responses API при наличии `OPENAI_API_KEY`.

Система должна:

- ориентироваться через `index → concepts → sources → raw`;
- делать сбор контекста проверяемым;
- сохранять пакет контекста как отдельный артефакт;
- запускать live-путь ответа через Responses API, если у студента есть
  `OPENAI_API_KEY`.

## Связь с курсом

- Память
- Контекст-инжиниринг
- RAG

## Стартовое состояние

- точка входа: текущий `main` после прохождения `M0`

До начала уже должно работать:

- `uv run --with pytest python -m pytest`
- `uv run kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault`
- `vault/index.md`
- `vault/log.md`
- `vault/wiki/index.md`

## Объём

Нужно сделать:

- собрать `vault/wiki/sources/` и `vault/wiki/concepts/`;
- выбирать wiki-страницы перед обращением к raw-заметкам;
- сохранять `artifacts/context/<run_id>.json`;
- сделать в трейсе видимыми этапы wiki-retrieval и сборки контекста;
- добавить live-проверку того же пути через `Responses API`.

Вне объёма:

- слой планирования
- embeddings / vector search
- обратная запись
- проверки состояния runtime-слоя
- hosted search, File Search или инструменты, которыми владеет модель

## Команды

- `uv run --with pytest python -m pytest`
- `uv run kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault`
- `OPENAI_API_KEY=... uv run --extra openai kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault --live-openai`, если есть API-ключ

## Ожидаемые артефакты

- `vault/wiki/index.md`
- `vault/wiki/sources/*.md`
- `vault/wiki/concepts/*.md`
- `artifacts/context/<run_id>.json`
- `artifacts/traces/<run_id>.jsonl`
- `vault/outputs/<run_id>.md`

## Критерии приёмки

- [ ] `uv run --with pytest python -m pytest` проходит
- [ ] `uv run kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault` проходит
- [ ] ответ собирается через путь `index → concepts → sources → raw`
- [ ] пакет контекста записан в `artifacts/context/`
- [ ] трейс показывает wiki retrieval и запись пакета контекста
- [ ] в `vault/wiki/` появились `sources/` и `concepts/`

Live-путь:

- [ ] если запускался live-путь, появился `artifacts/responses/<run_id>.json`

Этот пункт проверяет реальный OpenAI-first путь, но не нужен для CI без
API-ключа. В тестах live-путь покрывается тестовым клиентом Responses.

## Подсказки для отладки

- если `concepts/` не появились, проверь этап компиляции перед вопросом
- если пакет контекста пустой, проверь путь от wiki-страниц к raw-заметкам
- если ответ снова собирается напрямую из raw, проверь порядок retrieval-стадий

## Решение

- решение фиксируется текущим состоянием `main` и артефактами запуска

# M2

## Goal

В этом milestone мы добавляем `retrieval + planning` как inspectable pre-answer
layer.

Система должна:

- сначала собрать короткий план ответа;
- затем подобрать контекст под этот план;
- и только потом собрать grounded answer.

## Scope

Нужно сделать:

- добавить короткий plan artifact перед final answer;
- связать retrieval decisions с этим планом, а не только с вопросом;
- сделать видимым, какие части контекста были выбраны под каждый шаг плана;
- сохранить grounded final answer с provenance.

Out of scope:

- embeddings / vector search
- reranking zoo
- autonomous loops and self-reflection trees
- write-back в wiki
- health remediation
- eval harness

## Debugging Cues

- если plan artifact не появился, сначала проверь, создаётся ли он до answer generation
- если trace не содержит planning events, проверь, что planning layer пишет отдельные runtime events
- если answer не grounded, сравни plan, context packet и final citations
- если milestone начинает тянуть в embeddings или write-back, scope уже уехал

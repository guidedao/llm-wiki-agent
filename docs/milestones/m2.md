# M2

## Цель

В майлстоуне `M2` мы добавляем `retrieval + planning` как наблюдаемый слой
перед ответом.

Система должна:

- сначала собрать короткий план ответа;
- затем подобрать контекст под этот план;
- и только потом собрать ответ с опорой на источники.

## Связь с курсом

- `Память`
- `инженерия ретривала`
- `Планирование`

## Стартовое состояние

- точка входа: текущий `main` после прохождения `M1`
- до начала уже должно работать:
  - `uv run --with pytest python -m pytest`
  - `uv run kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault`
  - путь `index → concepts → sources → raw`
  - `artifacts/context/<run_id>.json`
  - `artifacts/traces/<run_id>.jsonl`

## Объём

Нужно сделать:

- добавить короткий артефакт плана перед финальным ответом;
- связать retrieval-решения с этим планом, а не только с вопросом;
- сделать видимым, какие части контекста были выбраны под каждый шаг плана;
- сохранить финальный ответ с опорой на источники и provenance.

Вне объёма:

- embeddings / vector search
- reranking zoo
- автономные петли и деревья self-reflection
- обратная запись в wiki
- исправление по health-сигналам
- eval-каркас

## Команды

- `uv run --with pytest python -m pytest`
- `uv run kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault`

## Ожидаемые артефакты

- `artifacts/plans/<run_id>.json`
- `artifacts/context/<run_id>.json`
- `artifacts/traces/<run_id>.jsonl`
- `vault/outputs/<run_id>.md`

## Критерии приёмки

- [ ] `uv run --with pytest python -m pytest` проходит
- [ ] `uv run kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault` проходит
- [ ] появился `artifacts/plans/<run_id>.json`
- [ ] в плане есть `2-4` шага, связанные с вопросом
- [ ] трейс содержит события вроде `plan_created` и `plan_completed`
- [ ] пакет контекста показывает, какой контекст был собран под план
- [ ] ответ остаётся привязанным к источникам и не смешивает retrieval с устойчивой памятью

## Подсказки для отладки

- если артефакт плана не появился, сначала проверь, создаётся ли он до генерации ответа
- если трейс не содержит событий планирования, проверь, что слой планирования пишет отдельные runtime-события
- если ответ не привязан к источникам, сравни план, пакет контекста и финальные цитаты
- если майлстоун начинает тянуть в embeddings или обратную запись, объём уже уехал

## Решение

- решение фиксируется текущим состоянием `main` и артефактами запуска

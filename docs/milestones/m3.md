# M3

## Goal

В этом milestone мы добавляем слой устойчивого выполнения поверх уже готового
пути `M2`.

Система должна показывать не только ответ, план и пакет контекста, но и то, как
конкретный запуск жил как объект: где он стартовал, чем завершился, какие
события оставил и проходит ли базовую проверку состояния.

## Scope

Нужно сделать:

- хранить run record как состояние запуска, а не только как случайный JSON;
- записывать terminal reason при успешном завершении;
- добавить health report для проверки ключевых артефактов запуска;
- держать `run_budget.yaml` как читаемый бюджет по стадиям;
- показать студенту, что run можно диагностировать по traces, artifacts и health.

Out of scope:

- distributed queue
- real background workers
- production leases
- write-back в wiki
- approval UI
- eval harness

- [ ] `run_budget.yaml` is present and describes the core stages

## Debugging Cues

- если health report failed, начните с missing checks внутри
  `artifacts/health/<run_id>.json`;
- если run не completed, сравните `artifacts/runs/<run_id>.json` и trace;
- если trace неполная, проверьте, на каком событии остановился запуск;
- если budget выглядит как pricing appendix, scope уехал: здесь нужен run-level
  budget по стадиям.

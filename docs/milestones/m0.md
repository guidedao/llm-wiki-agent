# M0

## Цель

В этой вехе мы делаем репозиторий живым сразу после первого запуска.

Система должна:

- оставаться детерминированной и локальной по умолчанию;
- собрать базовый `vault/` с `raw`, `wiki`, `outputs`, `index.md` и `log.md`;
- сгенерировать один ответ с опорой на источники;
- записать состояние запуска и трейс.

## Связь с курсом

- LLM
- API
- Инструменты
- Локальная база знаний

## Стартовое состояние

- точка входа: текущий `main` и этот milestone-док

До начала должно быть достаточно:

- `uv`
- `uv sync --frozen --extra dev`

## Объём

Нужно сделать:

- загрузить локальный markdown-корпус из `vault/raw/`;
- собрать `vault/wiki/index.md`;
- записать ответ с опорой на источники в `vault/outputs/`;
- сохранить состояние запуска и трейс;
- поддержать `vault/index.md` и `vault/log.md` как стабильные точки входа.

Вне объёма:

- live-веб-поиск
- платные адаптеры
- embeddings
- обратная запись
- runtime-governance

## Команды

- `uv sync --frozen --extra dev`
- `uv run --with pytest python -m pytest`
- `uv run kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault`

## Ожидаемые артефакты

- `vault/index.md`
- `vault/log.md`
- `vault/wiki/index.md`
- `vault/outputs/<run_id>.md`
- `artifacts/runs/<run_id>.json`
- `artifacts/traces/<run_id>.jsonl`

## Критерии приёмки

- [ ] `uv run --with pytest python -m pytest` проходит
- [ ] `uv run kb-agent --query-fixture fixtures/queries/m0_query.json --vault-root vault` проходит
- [ ] появился `vault/wiki/index.md`
- [ ] появился один файл ответа в `vault/outputs/`
- [ ] трейс записан в `artifacts/traces/`
- [ ] `vault/index.md` и `vault/log.md` созданы или обновлены

## Подсказки для отладки

- если `vault/` не собирается, сначала проверь существование `vault/raw/`
- если ответ не появился, проверь, не упал ли запуск до этапа записи output
- если трейс пустой, проверь вызовы слоя трейса вокруг demo-пути

## Решение

- решение фиксируется текущим состоянием `main` и артефактами запуска
